    <h1>Serialization of Custom Collections</h1>

    <h2>Why Serialize Custom Collections?</h2>
    <p>
        Serializing custom collections, such as <code>MyCollection</code>, is important for 
        preserving the state of complex data structures. This allows you to save and restore 
        collections of objects easily, making it convenient for data storage and transmission.
    </p>

    <h2>What We Will Do</h2>
    <p>
        We will implement serialization and deserialization for a custom collection class 
        called <code>MyCollection</code>, allowing us to store and retrieve the collection 
        of items.
    </p>

    <h2>How to Serialize a Custom Collection</h2>
    <p>
        We will use either <code>Pickle</code> or <code>JSON</code> to serialize the collection. 
        In this example, we will use <code>Pickle</code> for its ability to handle complex 
        Python objects.
    </p>

    <h2>Example Code</h2>
    <pre>
import pickle

class MyCollection:
    def __init__(self):
        self.items = []

    def add_item(self, item):
        self.items.append(item)

    def to_dict(self):
        return {'items': self.items}

    def save(self, filename):
        with open(filename, 'wb') as file:
            pickle.dump(self.to_dict(), file)

    @classmethod
    def load(cls, filename):
        with open(filename, 'rb') as file:
            data = pickle.load(file)
            collection = cls()
            collection.items = data['items']
            return collection

# Create a MyCollection instance
collection = MyCollection()
collection.add_item("Item 1")
collection.add_item("Item 2")

# Serialize the MyCollection object
collection.save('my_collection.pkl')

# Load the MyCollection object
loaded_collection = MyCollection.load('my_collection.pkl')
print(loaded_collection.items)  # Output: ['Item 1', 'Item 2']
